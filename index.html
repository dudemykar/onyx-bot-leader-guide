📖 Overview

Onyx Bot is the community's Discord assistant for the gaming guild. It runs leadership briefings, tracks public XP & Saga prestige, manages the roster (synced to a Google Sheet), detects and reminds about scheduled operations, analyzes screenshots with AI vision, and powers a diplomacy toolkit for the roundtable. It is built on Node.js / discord.js, uses the Anthropic Claude API, and persists everything to PostgreSQL on Railway.

🤖 AI models

  • claude-opus-4-8 — summaries, digests, plan detection, screenshot vision, diplomacy intents, roster screenshot reads, long-term memory
  • claude-sonnet-4-6 — /ask answers
  • claude-haiku-4-5 — intent classification & chill-zone banter

🔌 Integrations

  • Discord (discord.js) — the bot itself
  • Anthropic Claude API — all AI features
  • Google Sheets — roster source of truth & sync
  • PostgreSQL on Railway — saves flags, plans, XP, Saga, screenshots, RSVPs, memory, logs
🔑 Permission tiers

👑 Owner

The single owner account. Receives the 16:00 UTC personal digest and the automatic 25-message DM snapshots. Only the owner can run !summarize dm and !memory clear.

🛡️ Leadership / Admin

Members holding the leadership or admin role. Can run every ! command (summaries, flags, memory, activity, Saga awards/reset, utility). Saga claim-reactions are honored only from these roles.

📋 Roster managers

A hard-coded allow-list of Discord user IDs. Required for any roster mutation (add, remove, check, clear). Works only in the admin category / admin forum.

🎮 Members

Anyone holding a tracked member role. Earns XP, participates in Saga & RSVPs, and uses the player slash commands. /rank and /saga are usable by anyone, anywhere.

🎮 Public & player-facing features

📊 /rank EVERYONE

Purpose
Show the lifetime-XP leaderboard and a player's rank.
How to use
Run /rank for the top 20 with your own row pinned at the bottom, or /rank @player for one person's card.
Needs
None — usable by anyone, in any channel.
Example
/rank @Asaish
Result
An ephemeral (only-you) message with level, total XP, rank, and a progress bar to the next level.
Notes
Reply is visible only to the caller. Players with no XP yet are told so.

🏆 /saga EVERYONE

Purpose
Show the Saga Points prestige leaderboard and a player's standing.
How to use
/saga for the top 10 with your row pinned at the bottom, or /saga @player for one person's standing.
Needs
None — any channel.
Example
/saga
Result
Ephemeral leaderboard of Saga Points (rank · points). Only players with > 0 points appear.
Notes
Saga Points are awarded by leadership (see Progression); players cannot award their own.

❓ /ask TEAM CHANNELS

Purpose
Answer questions using community knowledge and the announcement forum threads.
How to use
/ask <question> in a team channel. It classifies intent and reads the relevant thread: latest announcement, objective, emergency, alliance/relations status, or a full "all details" recap.
Needs
Must be used in a team-category channel.
Example
/ask what is the latest objective?
Result
Ephemeral answer drawn only from the forum threads + game knowledge. If it can't find an answer, it says so rather than inventing one.
Notes
Answers reflect the four announcement forum threads; keep those current for best results.

📋 /roster & /roster-search TEAM CHANNELS

Purpose
View the official roster or check whether a player is on it.
How to use
/roster lists every member; /roster-search <name> looks one up (partial match).
Needs
Team-category channel.
Example
/roster-search morv
Result
Ephemeral list or a ✅/❌ match result.
Notes
Read-only for members. The roster is edited by roster managers (see Roster).

📅 Event reminders & RSVP buttons EVERYONE

Purpose
Notify the team ~1 hour before a scheduled operation and collect attendance.
How to use
No command — when a plan the bot detected is about to start, it posts a reminder with ✅ In / ❌ Out / ❔ Maybe buttons. Click one to RSVP; click again to change.
Needs
None to RSVP. A configured reminder role is optionally pinged.
Result
The post updates live with the In / Out / Maybe name lists. RSVPing also grants XP (once per event).
Notes
Reminders fire once per plan, only inside the final hour before start.

💬 Chill-zone banter PUBLIC

Purpose
Light, automatic community banter to keep the chill channel fun.
How to use
Just chat in the chill-zone channel. Roughly every 10 messages the bot drops one short witty reply.
Needs
None.
Result
An occasional one-liner from the bot (powered by Haiku). Rate-limited by a token so it never spams.
Notes
The separate "team joke" channel system exists in code but is currently turned OFF.

🏅 Leaderboard channels EVERYONE

Purpose
Always-visible rankings without running a command.
How to use
Open the team leaderboard channel (Saga Points + Lifetime XP, top 10 each) or the chill-zone leaderboard (Lifetime XP only).
Result
Two persistent messages the bot edits in place every 5 minutes (only when the data actually changed).
Notes
These channels earn no XP themselves (excluded).

ℹ️ !commands (team view) TEAM CHANNELS

Purpose
Show members the slash commands available to them.
How to use
Type !commands in a team channel.
Result
A self-deleting list of the player slash commands only.

In leadership/admin areas the same command shows the full leadership list instead.
🛡️ Leadership & admin commands

🧠 !summarize · !summarize all · !summarize dm LEADERSHIP

Purpose
Generate a structured intelligence brief from chat, plans, flags, and screenshot intel.
How to use
!summarize = this channel → leadership summary channel. !summarize all = all leadership channels + the four forum threads → leadership summary channel. !summarize dm = owner only; scans all leadership + diplomacy channels and DMs the owner a factual snapshot.
Needs
Leadership/admin category for the first two; owner for dm.
Example
!summarize all
Result
A teammate-style (or factual, for DM) brief. The 🔥 Tensions block is always split off and posted only to the salt-mines channel — never in the main brief.
Notes
Runs Claude Opus 4.8 at max effort. The owner can also DM the bot the text !summarize dm privately.

🖼️ !screenshots LEADERSHIP

Purpose
Collect the AI analyses of screenshots posted in a channel into one place.
How to use
Run !screenshots in a channel that has had images posted.
Result
Posts the stored screenshot descriptions to the persistent "📸 Screenshot Summaries" thread inside the leadership summary channel.
Notes
Screenshots are analyzed automatically on upload; this just surfaces them.

🧬 !memory · !memory clear LEADERSHIP

Purpose
View or reset the bot's long-term leadership memory (durable facts auto-distilled from the daily digests).
How to use
!memory shows the current fact list. !memory clear wipes it — owner only.
Needs
Leadership/admin category to view; owner to clear.
Result
A numbered list of durable facts, or confirmation it was cleared (it rebuilds from the next scheduled digest).
Notes
Leadership-only. Memory never includes tensions and never touches the roundtable or /ask.

🚩 !flag · !flags · !unflag LEADERSHIP

Purpose
Mark important messages or screenshots so they're highlighted in summaries.
How to use
!flag flags the previous message (or an attached/previous screenshot). !flags lists this channel's flags. !unflag [number] removes one.
Needs
Leadership/admin category.
Example
!unflag 2
Result
The flagged item is saved and appears in summaries within its time window; the list re-posts cleanly instead of stacking.

🏆 !award · !sagareset LEADERSHIP

Purpose
Grant Saga Points and run the seasonal reset.
How to use
!award @player <pathing|siege|final|number> [reason] grants points. !sagareset [label] archives standings, awards lifetime-XP bonuses to the top 10, then zeros everyone.
Needs
Leadership/admin role.
Example
!award @Lobo siege Great defense
Result
Points added (objective amounts come from game config) and the leaderboard updates. Reset announces the bonus list.
Notes
Objective types & point values are defined in the game config.

🕵️ !activity ADMIN AREA

Purpose
Leadership engagement report — who's showing up and who's gone quiet.
How to use
Run !activity in the admin category / admin forum.
Result
Scans the activity category (last ~30 days) and lists every role-holder with messages, reactions, RSVPs, and last-seen — split into Active and Quiet (7+ days silent).
Notes
Leadership intel only — never shown to members. Entirely separate from the public XP system.

🧹 !clearplans · !clearscreenshots LEADERSHIP

Purpose
Manually wipe stored plans or screenshot intel.
How to use
!clearplans empties the plan list. !clearscreenshots clears this channel's screenshots; !clearscreenshots all clears every channel's.
Needs
Leadership/admin category.
Result
Confirmation of how many items were removed; the change is persisted.

🏓 !ping · ℹ️ !commands (full) LEADERSHIP

Purpose
Quick health check and the complete command reference.
How to use
!ping confirms the bot is alive (leadership/admin only). !commands in a leadership/admin area prints the full command list.
Result
Self-deleting replies.
📋 Roster management

📋 !roster add / remove / clear ROSTER MANAGERS

Purpose
Maintain the official roster, kept in sync with the Google Sheet.
How to use
!roster add P1, P2, !roster remove P1, P2 (comma-separated), !roster clear. Works only in the admin category / admin forum.
Needs
A roster-manager user ID.
Example
!roster add Spyre, Rude
Result
Updates the in-bot roster, the roster thread post, and the Google Sheet (removed players are archived to a "Previous Players" tab).
Notes
!roster search <name> (no special permission) also works here.

🔎 !roster check ROSTER MANAGERS

Purpose
Compare an in-game roster screenshot against the bot roster to spot who left.
How to use
Attach a roster screenshot to !roster check (or post one, then run it). Up to 3 images are read.
Result
Claude vision extracts the names, then reports who is possibly missing (on bot roster, not in screenshot) and who is not on the bot roster.
Notes
Comparison is case-insensitive. Use it to reconcile the roster after membership changes.
🤝 Diplomacy & roundtable

🌍 Roundtable @mention toolkit ROUNDTABLE

Purpose
On-demand diplomacy reports generated from the roundtable conversation and the diplomacy thread.
How to use
In the roundtable channel, @mention the bot with one of: summarize, diplomacy, recent, agreements, report [team], flag, screenshots, help.
Result
The report is posted to the diplomacy summary channel (relationships use 🟢 allies / 🟡 neutral / 🔴 enemies).
Notes
Completely isolated from leadership data. @mentioning the bot in a non-roundtable diplomacy/friends channel returns a redirect.

📝 /summary DIPLOMACY / FRIENDS

Purpose
Quick AI recap of a diplomacy or friends channel.
How to use
Run /summary in a diplomacy/friends channel (not the roundtable — use the @mention there).
Result
An ephemeral summary of the channel's key points, future-dated events only.

📢 Persistent "Current Game Info" AUTOMATIC

Purpose
Keep the latest game info visible at the bottom of every diplomacy/friends channel.
How it works
Leadership posts/edits info in the info-feed channel; the bot mirrors that text as a fresh message in each diplomacy/friends channel and re-broadcasts when it changes.
Notes
Self-heals after restarts by deleting old copies before posting a new one.
⚙️ Automation & scheduled systems

These run on their own — no command needed.

🗓️ Scheduled digests DAILY · UTC

What it does
Four briefings per day: 06:00 (last 16h → leadership summary channel), 14:00 (last 8h → leadership summary channel), 16:00 (last 4h → owner DM, strict factual), and 14:00 roundtable (last 24h → diplomacy summary channel, fully isolated).
Notes
The three leadership digests also build & inject long-term memory. The roundtable digest never does. Failures DM the owner.

📨 25-message trigger AUTOMATIC

What it does
Counts human messages across all leadership + diplomacy channels (excluding chill-zone). At 25 it sends the owner a factual DM snapshot, then resets.
Notes
No public post — owner DM only. Tensions still route to salt-mines.

📅 Plan detection & reminders AUTOMATIC

What it does
Claude scans ops/leadership/diplomacy messages containing a weekday + month and extracts confirmed future plans (UTC-normalized). A timer pings ~1h before each with RSVP buttons. Expired plans are cleaned up hourly.
Notes
Only future, confirmed, "ours" plans are stored; enemy/speculative/past mentions are ignored. Duplicates are de-duped by date fingerprint.

🖼️ Screenshot analysis AUTOMATIC · AI VISION

What it does
Images posted in leadership, the screenshot-dump, the screenshots-only, and roundtable channels are read by Claude vision (maps, chat, stats) and stored. The intel flows into the relevant summaries.
Notes
Stored screenshots auto-expire after 48h or 25 per channel. Dump-channel intel feeds leadership summaries, never the roundtable.

🏅 Leaderboard auto-update EVERY 5 MIN

What it does
Edits the two persistent leaderboard messages in place — but only when XP or Saga data actually changed (a dirty flag avoids idle rewrites).

🏆 Saga claim reactions LEADERSHIP-GATED

What it does
In the claims channel, a leader reacting on a proof post with an objective emoji auto-awards that objective's points to the poster (guarded against double-awarding).
Notes
Only leadership/admin reactions count; objective emojis & values come from game config.
🫥 Silent / background systems

Running constantly with no visible output — important to know they exist.

📝 Persistent message log SILENT

What it does
Quietly records recent human messages from operational channels (leadership + diplomacy, roundtable, digest channels), capped at 200/channel, flushed to the database every 5 minutes and reloaded on startup.
Notes
Summaries fall back to this log if a live Discord fetch returns nothing (API hiccup or right after a restart). Casual chat is not logged.

🧬 Long-term memory SILENT · LEADERSHIP-ONLY

What it does
After each scheduled leadership digest, Claude distills durable facts (ongoing situations, standing decisions, alliances/wars, strategy), merges them into a capped 50-fact memory, and injects it into future digests for continuity.
Notes
Never includes tensions, never the roundtable, never /ask. View with !memory.

🕵️ Activity tracker SILENT · LEADERSHIP INTEL

What it does
Continuously tallies each member's messages, reactions, and RSVPs for the !activity report, flushing to the DB periodically.
Notes
Shares zero state with the public XP system — they are deliberately separate.

🔥 Tensions routing SILENT

What it does
Every summary (manual or automatic) splits off its 🔥 Tensions block and posts it only to the salt-mines channel — never in the main brief, never to the roundtable.
Notes
The bot names repeat troublemakers and does not soften this section.

🛡️ AI prompt hardening SILENT

What it does
Every Claude call carries anti-prompt-injection framing so user-submitted message/screenshot content can't hijack the bot's role.
Notes
A shared safe-extractor also prevents an empty/refused AI response from crashing a handler.
📈 Ranking & progression systems

⭐ Lifetime XP & leveling PUBLIC

How it's earned
Chatting (10 XP, 60-second per-user cooldown), emoji reactions (5 XP, deduped per message), and RSVPs (25 XP, once per event).
Curve
Three tiers — easy to L25, steeper to L100, brutal beyond. Never resets.
Gating
Only members with a tracked role earn XP, and excluded channels (leaderboard + screenshot-dump) earn none.
Notes
Level-ups announce in the channel where they happen. View via /rank.

🏆 Saga Points PUBLIC · LEADERSHIP-AWARDED

How it's earned
Proven in-game achievement only — leadership reacts with an objective emoji in the claims channel, or uses !award.
Season
!sagareset archives standings, awards lifetime-XP bonuses to the top 10 (5000/3000/2000 for the podium, 1000 for places 4–10), then zeros everyone.
Notes
Objectives & point values are defined in the game config. View via /saga.
🔌 Integrations

📗 Google Sheets (roster)

The roster's source of truth. On startup the bot loads members from the sheet; !roster edits write back to it, and removed players are archived to a "Previous Players" tab. Credentials come from an environment secret.

🗄️ PostgreSQL on Railway

Persists flags, plans, screenshots, RSVPs, activity, XP, Saga, the message log, and long-term memory as key→JSON rows. Loaded on startup; written by periodic flushes and on change.

🚢 Configuration & deployment

⚙️ Where settings live

  • config/constants.js — every channel/role/category ID, plus lists like the digest channels, XP-tracked roles, excluded channels, salt-mines, screenshot-dump, owner ID, reminder role
  • config/game.js — the game world: factions, teams, members, alliance standings, and Saga objectives/points/emojis
  • config/rosterManagers.js — who may edit the roster

📦 Deploy

Edit → push to GitHub → Railway auto-deploys → watch logs for Bot is online as …. Secrets (.env, Google credentials) are gitignored and set in Railway. Run node register-roster.js once after adding or removing a slash command.
